Truthy vs falsy

Function basics

Remember that there are only 7 values in JavaScript that are falsy:

false
0
0n: 0 as a BigInt
'': Empty string
null
undefined
NaN
Every other value is truthy.

If you use == to compare values Javascript will helpfully attempt to convert operands before doing compare
"5" == 5 is true
"4" == 5 is false
But
"5" === 5 is false because no conversion is done.

Always use ===

?? vs ||

Nullish coalescing operator

The result of a ?? b is:

if a is defined, then a,
if a isnâ€™t defined, then b.

The result of a || b is:

if a is falsy then b,
if a is truthy, then a

Function declaration vs Function expression

Function expressions

This syntax is called a Function Declaration:

function sayHi() {
  alert( "Hello" );
}

There is another syntax for creating a function that is called a Function Expression.

let sayHi = function() {
  alert( "Hello" );
};

In either case say sayHi() will call the function.

Primitives vs objects
Javascript object properties are stored with the object.
	{
		p1:5, 
		p2:6; 
		p3:function(){console.log("in p3")
	}

Javascript primitives have no properties stored with them yet properties can be called on them, e.g. 
"hello".toUpperCase()

This is done by using a temporary object wrapper, e.g., 

let str = "Hello";
alert( str.toUpperCase() ); // HELLO

is equivalent to 

let str = "hello";
str = new String(str);	// Temporary object wrapper
alert( str.toUpperCase() ); // HELLO
str = "hello"; 			// Remove temporary object wrapper
